# ImageOrganizer 环境配置说明

## 1. 项目简介

本项目是一个基于 Python 的图片与视频整理工具。
程序会扫描源目录，优先读取图片的 EXIF 时间；如果没有 EXIF 时间，则使用文件修改时间。
然后按照“年\月\日”的目录结构，将文件移动或拷贝到目标目录。


## 2. 推荐环境

- 操作系统：Windows 10 或 Windows 11
- Python：3.10 或更高版本
- 终端：PowerShell 或命令提示符（CMD）

本项目是本地脚本工具，不依赖数据库、Web 服务或外部接口。


## 3. 必要依赖

本项目当前依赖以下第三方 Python 库：

- `Pillow`

`Pillow` 用于读取图片的 EXIF 元数据。


## 4. 创建虚拟环境

在项目根目录打开终端后，执行：

```powershell
python -m venv venv
```

这条命令会在当前项目根目录下创建一个名为 `venv` 的文件夹。

例如，如果你的项目放在：

```text
D:\ImageOrganizer
```

那么虚拟环境默认会创建在：

```text
D:\ImageOrganizer\venv
```

其中常用位置如下：

- 虚拟环境目录：`D:\ImageOrganizer\venv`
- 虚拟环境中的 Python：`D:\ImageOrganizer\venv\Scripts\python.exe`
- 虚拟环境中的 pip：`D:\ImageOrganizer\venv\Scripts\pip.exe`

也就是说，本项目安装到虚拟环境中的依赖库，不会装到 `main.py` 所在目录里，而是装到这个虚拟环境自己的 Python 目录中。
只要你使用的是 `venv\Scripts\python.exe` 或先激活了这个 `venv`，依赖就会从这个虚拟环境里读取。

在 PowerShell 中激活虚拟环境：

```powershell
.\venv\Scripts\Activate.ps1
```

如果你使用的是命令提示符（CMD）：

```cmd
venv\Scripts\activate.bat
```

激活成功后，终端前面通常会出现类似 `(venv)` 的提示，表示当前已经进入项目自己的虚拟环境。

如果你不确定自己激活的是不是这个项目下的虚拟环境，可以执行：

```powershell
Get-Command python
```

或：

```powershell
python -c "import sys; print(sys.executable)"
```

输出结果应当指向：

```text
你的实际项目路径\venv\Scripts\python.exe
```


## 5. 安装依赖

激活虚拟环境后，执行：

```powershell
pip install Pillow
```

上面的安装命令会把 `Pillow` 安装到当前已激活的虚拟环境中。

如果你担心 `pip` 装错位置，建议使用更明确的写法：

```powershell
.\venv\Scripts\python.exe -m pip install Pillow
```

这样可以明确保证：依赖被安装到当前项目的 `venv` 中，而不是系统全局 Python 或别的虚拟环境中。

可选检查命令：

```powershell
pip show Pillow
```

更明确的检查方式：

```powershell
.\venv\Scripts\python.exe -m pip show Pillow
```


## 6. 验证环境

检查 Python 和 pip：

```powershell
python --version
pip --version
```

检查 Pillow 是否安装成功：

```powershell
python -c "from PIL import Image; print('Pillow OK')"
```

如果你想彻底避免“当前终端到底用了哪个 Python”的问题，建议改用完整路径验证：

```powershell
.\venv\Scripts\python.exe -c "from PIL import Image; print('Pillow OK')"
```


## 7. 项目结构说明

主要目录和文件如下：

- `main.py`
  程序入口文件
- `core/`
  EXIF 读取和日期识别逻辑
- `services/`
  文件整理逻辑
- `locales/`
  多语言提示资源，当前支持 `zh`、`en`、`ja`
- `log/`
  每次运行后生成的日志目录


## 8. 运行脚本

请务必注意：`main.py` 位于项目根目录中，因此应当先进入项目根目录，再执行运行命令。

例如：

```powershell
cd D:\ImageOrganizer
```

如果你已经激活了当前项目的 `venv`，可以这样运行：

```powershell
python .\main.py --src D:\InputPhotos --dst D:\SortedPhotos
```

如果你不想依赖“激活是否成功”，推荐直接使用虚拟环境里的完整 Python 路径运行：

```powershell
.\venv\Scripts\python.exe .\main.py --src D:\InputPhotos --dst D:\SortedPhotos
```

这是一种更明确、也更不容易出错的执行方法。它可以同时避免以下问题：

- 终端当前不在项目根目录，导致找不到 `main.py`
- 当前用的不是项目的 `venv`，导致找不到 `Pillow`
- 忘记了 `venv` 在哪里

基本示例：

```powershell
.\venv\Scripts\python.exe .\main.py --src D:\InputPhotos --dst D:\SortedPhotos
```

拷贝模式示例：

```powershell
.\venv\Scripts\python.exe .\main.py --src D:\InputPhotos --dst D:\SortedPhotos --mode copy
```

日语界面示例：

```powershell
.\venv\Scripts\python.exe .\main.py --src D:\InputPhotos --dst D:\SortedPhotos --lang ja
```

推荐顺序如下：

1. 先 `cd` 到项目根目录
2. 再确认 `venv` 在 `.\venv`
3. 尽量使用 `.\venv\Scripts\python.exe .\main.py ...` 运行

这样最容易排查路径和依赖问题。


## 9. 运行参数说明

- `--src`
  源目录，必填
- `--dst`
  目标目录，必填
- `--mode`
  可选 `move` 或 `copy`，默认值为 `move`
- `--lang`
  可选 `zh`、`en`、`ja`，默认值为 `zh`


## 10. 输出与日志

程序会自动在项目目录下创建 `log` 文件夹。

每次运行会生成一个日志文件，格式如下：

```text
organize_log_YYYYMMDD_HHMMSS.txt
```

程序执行完成后，会在终端输出本次日志文件的完整路径。


## 11. 常见环境问题

### Python 无法识别

如果终端提示找不到 `python`，请确认：

- 已安装 Python
- Python 已加入系统 `PATH`

### PowerShell 无法激活虚拟环境

如果 PowerShell 阻止脚本执行，可先运行：

```powershell
Set-ExecutionPolicy -Scope Process RemoteSigned
```

然后重新执行激活命令。

### Pillow 未安装或导入失败

如果出现 `PIL` 相关错误，请执行：

```powershell
.\venv\Scripts\python.exe -m pip install --upgrade Pillow
```

如果仍不确定当前环境是否正确，可执行：

```powershell
python -c "import sys; print(sys.executable)"
.\venv\Scripts\python.exe -m pip show Pillow
```


## 12. 推荐使用流程

1. 在项目根目录打开终端，例如：`cd D:\ImageOrganizer`
2. 执行 `python -m venv venv`，确认虚拟环境创建在 `你的项目路径\venv`
3. 优先使用 `.\venv\Scripts\python.exe -m pip install Pillow` 安装依赖
4. 使用 `.\venv\Scripts\python.exe .\main.py --src ... --dst ...` 运行脚本
5. 执行完成后查看 `log` 目录中的日志文件
