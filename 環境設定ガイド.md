# ImageOrganizer 環境設定ガイド

## 1. プロジェクト概要

本プロジェクトは、Python ベースの画像・動画整理ツールです。
ソースフォルダを走査し、画像の EXIF 日時を優先して取得します。
EXIF 日時がない場合は、ファイルの更新日時を使用します。
その後、ファイルを「年\月\日」のフォルダ構成で移動またはコピーします。


## 2. 推奨環境

- OS: Windows 10 または Windows 11
- Python: 3.10 以上
- ターミナル: PowerShell またはコマンドプロンプト（CMD）

本プロジェクトはローカル実行用のスクリプトであり、
データベース、Web サービス、外部 API は不要です。


## 3. 必要な依存ライブラリ

本プロジェクトで使用する主なサードパーティ製 Python ライブラリは次のとおりです。

- `Pillow`

`Pillow` は画像の EXIF メタデータを読み取るために使用されます。


## 4. 仮想環境の作成

プロジェクトのルートディレクトリで以下を実行してください。

```powershell
python -m venv venv
```

PowerShell で仮想環境を有効化する場合：

```powershell
.\venv\Scripts\Activate.ps1
```

コマンドプロンプト（CMD）を使用する場合：

```cmd
venv\Scripts\activate.bat
```


## 5. 依存関係のインストール

仮想環境を有効化した後、以下を実行してください。

```powershell
pip install Pillow
```

確認用コマンド：

```powershell
pip show Pillow
```


## 6. 環境確認

Python と pip の確認：

```powershell
python --version
pip --version
```

Pillow の動作確認：

```powershell
python -c "from PIL import Image; print('Pillow OK')"
```


## 7. プロジェクト構成

主なファイルとフォルダ：

- `main.py`
  実行エントリーポイント
- `core/`
  EXIF 読み取りと日付判定ロジック
- `services/`
  ファイル整理ロジック
- `locales/`
  多言語メッセージリソース。現在は `zh`、`en`、`ja` に対応
- `log/`
  実行ごとのログ出力先


## 8. スクリプトの実行

基本例：

```powershell
python main.py --src D:\InputPhotos --dst D:\SortedPhotos
```

コピーモードの例：

```powershell
python main.py --src D:\InputPhotos --dst D:\SortedPhotos --mode copy
```

日本語表示の例：

```powershell
python main.py --src D:\InputPhotos --dst D:\SortedPhotos --lang ja
```


## 9. 実行パラメータ

- `--src`
  ソースフォルダ。必須
- `--dst`
  保存先フォルダ。必須
- `--mode`
  `move` または `copy`。デフォルトは `move`
- `--lang`
  `zh`、`en`、`ja` を指定可能。デフォルトは `zh`


## 10. 出力とログ

プログラムはプロジェクトディレクトリ配下に `log` フォルダを自動作成します。

各実行ごとに次の形式のログファイルが生成されます。

```text
organize_log_YYYYMMDD_HHMMSS.txt
```

実行完了後、ターミナルにログファイルのフルパスが表示されます。


## 11. よくある環境トラブル

### Python が認識されない

`python` コマンドが見つからない場合は、以下を確認してください。

- Python がインストールされていること
- Python がシステムの `PATH` に追加されていること

### PowerShell で仮想環境を有効化できない

PowerShell の実行ポリシーによりブロックされる場合は、先に以下を実行してください。

```powershell
Set-ExecutionPolicy -Scope Process RemoteSigned
```

その後、再度有効化コマンドを実行してください。

### Pillow のインポートに失敗する

`PIL` 関連のエラーが出る場合は、以下を実行してください。

```powershell
pip install --upgrade Pillow
```


## 12. 推奨利用手順

1. プロジェクトのルートディレクトリでターミナルを開く
2. 仮想環境を作成して有効化する
3. `Pillow` をインストールする
4. `--src` と `--dst` を指定してスクリプトを実行する
5. 実行後に `log` フォルダ内のログを確認する
