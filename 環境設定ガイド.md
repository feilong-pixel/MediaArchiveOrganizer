# Media Archive Organizer 環境設定ガイド

## 1. プロジェクト概要

本プロジェクトは、Python ベースの画像・動画整理ツールです。
ソースフォルダを走査し、画像の EXIF 日時を優先して取得します。
EXIF 日時がない場合は、ファイルの更新日時を使用します。
その後、ファイルを「年\月\日」のフォルダ構成で移動またはコピーします。
重複検出が有効な場合でも、重複ファイルは通常の日付アーカイブフォルダに配置され、`保持ファイル名_dupN.ext` の形式でリネームされます。
また、重複関係は `duplicate_report.csv` に追記されます。


## 2. 推奨環境

- OS: Windows 10 または Windows 11
- Python: 3.10 以上
- ターミナル: PowerShell またはコマンドプロンプト（CMD）

本プロジェクトはローカル実行用のスクリプトであり、
データベース、Web サービス、外部 API は不要です。


## 3. 必要な依存ライブラリ

本プロジェクトで使用する主なサードパーティ製 Python ライブラリは次のとおりです。

- `Pillow`
- `exifread`
- `pywin32`（Windows のみ）
- `pytest`（任意。テスト実行時のみ必要）

`Pillow` は画像の EXIF メタデータを読み取るために使用されます。
`exifread` は、Pillow では取得できない追加の EXIF 情報やメーカー固有のメタデータ（MakerNote など）を読み取るために使用します。より正確な撮影日時の取得に役立ちます。
`pywin32` は Windows 環境でファイルの作成日時・アクセス日時・更新日時を保持するために使用します。`move` や `copy` モードでファイルを操作する際に、元のタイムスタンプを失わないようにするために必要です。
`pytest` は自動テストの実行に使用されます。


## 4. 仮想環境の作成

プロジェクトのルートディレクトリで以下を実行してください。

```powershell
python -m venv venv
```

このコマンドを実行すると、現在のプロジェクトルート配下に `venv` というフォルダが作成されます。

たとえば、プロジェクトを次の場所に置いている場合：

```text
D:\ImageOrganizer
```

仮想環境は次の場所に作成されます。

```text
D:\ImageOrganizer\venv
```

よく使う場所は次のとおりです。

- 仮想環境フォルダ: `D:\ImageOrganizer\venv`
- 仮想環境内の Python: `D:\ImageOrganizer\venv\Scripts\python.exe`
- 仮想環境内の pip: `D:\ImageOrganizer\venv\Scripts\pip.exe`

つまり、依存ライブラリは `main.py` と同じ場所にインストールされるのではなく、この `venv` 内の Python 環境にインストールされます。
`venv\Scripts\python.exe` を使うか、この `venv` を有効化している限り、依存関係はこの仮想環境から読み込まれます。

PowerShell で仮想環境を有効化する場合：

```powershell
.\venv\Scripts\Activate.ps1
```

コマンドプロンプト（CMD）を使用する場合：

```cmd
venv\Scripts\activate.bat
```

有効化に成功すると、通常はターミナルの先頭に `(venv)` のような表示が付きます。

今使っている Python がこのプロジェクトの仮想環境か確認したい場合は、次を実行してください。

```powershell
Get-Command python
```

または：

```powershell
python -c "import sys; print(sys.executable)"
```

結果は次のような場所を指しているはずです。

```text
実際のプロジェクトパス\venv\Scripts\python.exe
```


## 5. 依存関係のインストール

仮想環境を有効化した後、以下を実行してください。

```powershell
pip install Pillow
```

このコマンドは、現在有効になっている仮想環境に `Pillow` をインストールします。

誤った Python 環境にインストールされるのを避けたい場合は、より明示的な次の書き方を推奨します。

```powershell
.\venv\Scripts\python.exe -m pip install Pillow exifread pywin32
```

テストも実行したい場合は、次を使用してください。

```powershell
.\venv\Scripts\python.exe -m pip install Pillow pytest
```

これにより、依存関係がシステム全体の Python や別の仮想環境ではなく、このプロジェクトの `venv` に確実にインストールされます。

確認用コマンド：

```powershell
pip show Pillow
```

より明示的な確認方法：

```powershell
.\venv\Scripts\python.exe -m pip show Pillow exifread pywin32
```


## 6. 環境確認

Python と pip の確認：

```powershell
python --version
pip --version
```

Pillow の動作確認：

```powershell
python -c "from PIL import Image; print('Pillow OK')"
```

どの Python を使っているか曖昧にしたくない場合は、次の確認方法を推奨します。

```powershell
.\venv\Scripts\python.exe -c "from PIL import Image; print('Pillow OK')"
```


## 7. プロジェクト構成

主なファイルとフォルダ：

- `main.py`
  実行エントリーポイント
- `core/`
  EXIF 読み取りと日付判定ロジック
- `services/`
  ファイル整理ロジック
- `locales/`
  多言語メッセージリソース。現在は `zh`、`en`、`ja` に対応
- `log/`
  実行ごとのログ出力先
- `tests/`
  整理処理と重複処理を確認する自動テスト


## 8. スクリプトの実行

`main.py` はプロジェクトルートにあるため、先にプロジェクトルートへ移動してから実行してください。

例：

```powershell
cd D:\ImageOrganizer
```

このプロジェクトの `venv` をすでに有効化している場合は、次のように実行できます。

```powershell
python .\main.py --src D:\InputPhotos --dst D:\SortedPhotos
```

有効化の成否に依存したくない場合は、仮想環境内の Python をフルパスで指定する方法を推奨します。

```powershell
.\venv\Scripts\python.exe .\main.py --src D:\InputPhotos --dst D:\SortedPhotos
```

この方法はより明確で、次のような問題を避けやすくなります。

- ターミナルの現在位置がプロジェクトルートではなく、`main.py` が見つからない
- 現在使っている Python がこのプロジェクトの `venv` ではなく、`Pillow` が見つからない
- `venv` がどこにあるか分からなくなる

基本例：

```powershell
.\venv\Scripts\python.exe .\main.py --src D:\InputPhotos --dst D:\SortedPhotos
```

コピーモードの例：

```powershell
.\venv\Scripts\python.exe .\main.py --src D:\InputPhotos --dst D:\SortedPhotos --mode copy
```

日本語表示の例：

```powershell
.\venv\Scripts\python.exe .\main.py --src D:\InputPhotos --dst D:\SortedPhotos --lang ja
```

推奨手順は次のとおりです。

1. まずプロジェクトルートへ `cd` する
2. `venv` が `.\venv` にあることを確認する
3. 実行時は `.\venv\Scripts\python.exe .\main.py ...` を優先する

この手順にすると、パスや依存関係の問題を切り分けやすくなります。


## 9. 実行パラメータ

- `--src`
  ソースフォルダ。必須
- `--dst`
  保存先フォルダ。必須
- `--mode`
  `move` または `copy`。デフォルトは `move`
- `--duplicate-detection`
  `off`、`phash`、`strict` を指定可能。デフォルトは `phash`
- `--phash-threshold`
  `phash` 使用時だけ有効な整数しきい値。デフォルトは `4`
- `--lang`
  `zh`、`en`、`ja` を指定可能。デフォルトは `en`


## 10. 出力とログ

プログラムはプロジェクトディレクトリ配下に `log` フォルダを自動作成します。

各実行ごとに次の形式のログファイルが生成されます。

```text
organize_log_YYYYMMDD_HHMMSS.txt
```

実行完了後、ターミナルにログファイルのフルパスが表示されます。

重複が検出された場合は、同じフォルダに次の CSV も追記されます。

```text
duplicate_report.csv
```

現在の CSV には少なくとも次の項目が含まれます。

- `original_name`
- `original_path`
- `kept_path`
- `duplicate_method`
- `hash`
- `duplicate_path`


## 11. よくある環境トラブル

### Python が認識されない

`python` コマンドが見つからない場合は、以下を確認してください。

- Python がインストールされていること
- Python がシステムの `PATH` に追加されていること

### PowerShell で仮想環境を有効化できない

PowerShell の実行ポリシーによりブロックされる場合は、先に以下を実行してください。

```powershell
Set-ExecutionPolicy -Scope Process RemoteSigned
```

その後、再度有効化コマンドを実行してください。

### Pillow のインポートに失敗する

`PIL` 関連のエラーが出る場合は、以下を実行してください。

```powershell
.\venv\Scripts\python.exe -m pip install --upgrade Pillow
```

現在の環境が正しいか不安な場合は、次も確認してください。

```powershell
python -c "import sys; print(sys.executable)"
.\venv\Scripts\python.exe -m pip show Pillow exifread pywin32
```


## 12. 推奨利用手順

1. プロジェクトルートでターミナルを開く。例: `cd D:\ImageOrganizer`
2. `python -m venv venv` を実行し、仮想環境が `実際のプロジェクトパス\venv` に作成されたことを確認する
3. 依存関係のインストールには `.\venv\Scripts\python.exe -m pip install Pillow` を優先する
4. 実行時は `.\venv\Scripts\python.exe .\main.py --src ... --dst ...` を使う
5. 実行後に `log` フォルダ内のログを確認する

テストを実行する場合は、次を使用してください。

```powershell
.\venv\Scripts\python.exe -m pytest -q
```
